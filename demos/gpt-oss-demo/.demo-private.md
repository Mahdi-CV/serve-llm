# PRIVATE - personal cheat sheet

Active demo for gpt-oss 120B on the MI350 box.

## What is set up

- Single general-purpose container `vllm-openai-rocm-ctr` (idle, `--init`) runs as
  standing infra on the MI350 server. Stays up between takes.
- Serving = launching `vllm serve openai/gpt-oss-120b` inside it via `docker exec`.
- vLLM compile cache mounted from host at `/data/vllm_cache` (persists across kills).
- Reset between takes: `./stage.sh` kills the model process, container stays up.
- The old per-model container `vllm-gpt-oss-120b` is stopped by stage.sh on first run.

## The prompt to say on camera

> run gpt-oss 120B on do-mi350-node with vLLM and give me the endpoint

## Flow (every take)

1. Stage the box:

       cd ~/gpt-oss-demo && ./stage.sh

2. Optional live server view in a second terminal:

       ./watch.sh
   or SSH in manually:
       ssh do-mi350-node
       ./check_docker_log.sh

3. Open Claude:

       claude --dangerously-skip-permissions

4. Say the prompt. Endpoint + table in ~2-4 min (warm path with TP=8, 120B model).

5. Point hermes at it:

       ssh do-mi350-node
       ./set_hermes_custom_model.sh
       hermes chat

## Reset between takes

Run `./stage.sh` again. It kills the model process; container stays up.

## Timing

- First launch with new generic container (no vllm_cache yet): longer, cache gets
  populated at `/data/vllm_cache`. Do this well before the demo.
- Warm relaunch after a process kill: compile cache is on disk; faster. This is the
  demo path.
- TP=8 + 120B: expect more warmup time than the MI300X single-GPU Qwen demo.
  Time this in rehearsal.

## Backup

If anything fails, the old `vllm-gpt-oss-120b` container can be restarted:
  `ssh do-mi350-node 'docker start vllm-gpt-oss-120b'`
It uses the exact same flags that worked before. Port 8000.

## What is honest to say if asked

- Weights are cached on the box (183 GB).
- We keep a general-purpose vLLM container running as infra; serving starts the model
  inside it. The wait is the engine warming up on 8 GPUs (HBM load, graph compile/replay,
  KV cache allocation across TP=8).
- A cold box takes longer; we pre-stage to stay in budget.
